# tasks.md テンプレート

`<workdir>/tasks.md` の構成テンプレート。**参照による自己完結**を原則とする: 各タスクはタスク固有情報のみを持ち、仕様の詳細は仕様文書 spec.md の要件 ID・節への参照で解決する。仕様の本文を転記しない。

```markdown
# <作業単位名> — 実装タスク

> 仕様の詳細は同じディレクトリの仕様文書 spec.md を参照する。
> このファイルには仕様を転記しない。

## File Structure Plan

| ファイルパス | 区分 | 責務                                      |
| ------------ | ---- | ----------------------------------------- |
| `src/...`    | 新規 | ...                                       |
| `src/...`    | 変更 | ...                                       |
| `src/...`    | 削除 | ...(置換・廃止時。使用ゼロ確認を経て削除) |

## タスク一覧

- [ ] 1. <メインタスクの説明> (P)
  - [ ] 1.1 <サブタスクの説明>
        _Requirements: 1.1, 1.2_
        _Boundary: <Component>_
    - 対象ファイル: `src/...`(新規/変更), `tests/...`
    - 仕様参照: spec.md §5 <インターフェース名>
    - 検証コマンド: `<ビルド/テスト/リント>`
  - [ ] 1.2 <サブタスクの説明>
        _Requirements: 1.3_
        _Boundary: <Component>_
        _Depends: 1.1_
        _Knowledge: <トピック>_
    - 対象ファイル: ...
    - 仕様参照: ...
    - 検証コマンド: ...

- [ ] 2. <メインタスクの説明>
  - [ ]* 2.1 <テストを後回しにするタスク>
    _Requirements: 2.1_
    _Boundary: <Component>_
    - 対象ファイル: ...
    - 仕様参照: ...
    - 検証コマンド: ...

## Implementation Notes

(このセクションは dev-implement が実装中の学習・選択した知識 port・横断的な気付きを追記する領域。初期は空でよい)
```

## 注記の意味(正本: `../../dev-core/references/notation.md`)

| 注記                  | 意味                                                                         |
| --------------------- | ---------------------------------------------------------------------------- |
| `(P)`                 | 他タスクと並行実行可能(境界が独立)                                           |
| `- [ ]*`              | テストを後回しにするタスク                                                   |
| `_Requirements: X.Y_` | このタスクがカバーする要件 ID(数値のみ)                                      |
| `_Boundary: Name_`    | このタスクが所有するコンポーネント                                           |
| `_Depends: X.Y_`      | 境界を越える依存(先行すべきタスク)                                           |
| `_Knowledge: name_`   | 知識 port の明示上書き(通常は書かない。特定 port の注入を固定したいときのみ) |

## ルール(要約)

- 階層番号: メイン `1.`、サブ `1.1`。
- すべての要件 ID をいずれかのタスクでカバーする。
- 仕様の詳細を転記しない。実装者は `_Requirements:_` の ID と仕様参照で spec.md を読む。
- `(P)` は境界が真に独立する場合のみ。
- TDD: テストファースト、または各タスクで RED→GREEN を想定。
- 条件分岐でエラー伝播・挙動が分かれる箇所は、各分岐に個別のテスト割当をする。
- ドキュメント反映(README 等)は独立サブタスクとして明示する。
